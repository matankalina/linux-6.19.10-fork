# TCP Receive Instrumentation Design

This document describes the instrumentation added to the Linux TCP stack to track payload bytes received by the kernel before they are consumed by the application.

The goal is to make every change in unread TCP payload explainable.

# Buffered State Definition

Unread payload is defined as:

total_unread_bytes =
readable_queue_bytes + ofo_queue_bytes

Where:

readable_queue_bytes

Payload currently stored in `sk_receive_queue`.

ofo_queue_bytes

Payload stored in `tp->out_of_order_queue`.

---

# Validation Strategy

The instrumentation is validated using packet captures.

Traffic is captured using:

tcpdump  
Wireshark

The following properties must hold:

1. Every payload byte observed on the wire appears in a receive event.
2. Buffered unread bytes match the internal TCP queue state.
3. Promotion from OFO to readable does not change the total buffered payload.
4. Duplicate packets are recorded as events but do not increase buffered state.
